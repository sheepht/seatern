# 婚禮座位安排系統 PRD

> 版本：3.0  
> 日期：2026-03-23  
> 核心原則：**不要讓賓客多做事**

---

## 1. 產品目標

幫助新人快速完成婚禮座位安排，解決以下痛點：

| 痛點 | 解法 |
|-----|------|
| 分類沒做好，同群組的人被拆散 | 依分類自動分桌 |
| 有些人絕對不能坐一起 | 新人標記「不想同桌」 |
| 群組人數超過一桌 | 自動拆分到相鄰桌 |
| RSVP 人數一直變 | 新人直接在系統改，重新分桌 |
| 用 Excel 改來改去很煩 | 拖曳調整 + 即時滿意度回饋 |

---

## 2. 核心原則

### 不要讓賓客多做事

賓客只需要填**原本就會填的 Google 表單**：

```
必填：
├── 姓名
├── 分類（大學同學/公司同事/家人/...）
└── 出席人數

選填：
├── 暱稱
├── 想同桌（最多 3 位）← 選填，不填也沒關係
└── 備註（素食等）
```

所有複雜的事情都由**新人在系統裡處理**：
- 誰跟誰想同桌 → 新人標記
- 誰跟誰不能同桌 → 新人標記
- 人數變動 → 新人直接改
- 座位微調 → 新人拖曳

## 6. 分類系統

分類是自動分桌的**主要依據**。

### 預設分類

```
男方：家人、大學同學、高中同學、公司同事、其他朋友
女方：家人、大學同學、高中同學、公司同事、其他朋友
共同：共同朋友
其他：未分類
```

### 自訂分類

新人可以新增，例如「健身房朋友」「社團學長姐」。

---

## 7. 自動分桌邏輯

### 硬約束（必須遵守）

- 「不想同桌」的人絕對不能同桌
- 每桌不超過設定人數

### 軟約束（優先順序）

1. 「想同桌」的人優先同桌
2. 同分類的人優先同桌
3. 人數平衡
4. 群組超過一桌時，拆到相鄰桌

### 違規警告

如果有無法滿足的硬約束（例如 A 不想跟 B 同桌，但 B 不想跟 C 同桌，A 又想跟 C 同桌），系統顯示警告讓新人手動處理。

---

## 8. 滿意度分數

讓新人知道安排得好不好。

### 計算方式

```
個人滿意度 = 基礎分(50) + 分類分(0-25) + 關係分(0-25)

分類分：同桌有多少比例是同分類
關係分：想同桌的人有多少在同桌
```

### 顯示

| 分數 | 顏色 |
|-----|------|
| 80+ | 🟢 綠色 |
| 60-79 | 🟡 黃色 |
| <60 | 🔴 紅色 |

- 桌平均滿意度
- 全場平均滿意度

## 11. Google Sheet 匯入

### 欄位對應

| 表單欄位 | 系統欄位 |
|---------|---------|
| 姓名 / 名字 / Name | name |
| 暱稱 / 小名 | nickname |
| 分類 / 類別 / 群組 | category |
| 出席人數 / 人數 | attendeeCount |
| 想同桌 / 同桌 | wantToSitWith |
| 備註 / Note | note |

### 匯入方式

貼上 Google Sheet 公開連結，系統自動解析。

### 同步邏輯：只新增，不覆蓋

```
比對姓名：
├── Sheet 有，系統沒有 → 新增
├── 兩邊都有 → 跳過（不動）
└── Sheet 沒有，系統有 → 跳過（不刪）
```

**為什麼這樣設計：**

- 新人在系統裡改過的資料（人數、座位、關係標記）不會被蓋掉
- 新人對資料有完全掌控權
- 同步只用來「撈新賓客」

**使用情境：**

```
賓客陸續填表 → Google Sheet 累積資料
        │
        ▼
新人匯入系統（第一次）
        │
        ▼
新人在系統裡調整（改人數、標記關係、調座位）
        │
        ▼
又有新賓客填表了
        │
        ▼
新人再同步一次 → 只新增新賓客，舊資料不動
```

### 同步完成提示

```
┌─────────────────────────────────────┐

│                                     │
```
